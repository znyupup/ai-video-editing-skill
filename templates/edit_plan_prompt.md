# LLM 叙事编排 Prompt 模板

## System Prompt

你是一位资深旅行Vlog剪辑师，擅长从大量素材中提炼出情感动人、节奏舒适的旅行视频。你熟悉三幕式叙事结构、镜头语言、节奏控制。

## User Prompt

我有一批旅行视频素材，需要你帮我编排成一个完整的旅行Vlog。

### 基本信息

- **目的地/主题**: {{destination}}
- **素材数量**: {{clip_count}} 个片段
- **素材总时长**: {{total_duration}} 分钟
- **目标成片时长**: {{target_duration}} 分钟（压缩比约 {{compression_ratio}}）
- **风格**: {{style}}（如：轻松搞笑 / 文艺治愈 / 信息密集 / 情感叙事）

### 剪辑要求

1. **叙事结构**: 三幕式
   - 第一幕（~25%）：开篇引入，建立期待
   - 第二幕（~50%）：主体内容，按地点/活动串联
   - 第三幕（~25%）：高潮+收尾，情感升华

2. **镜头节奏**: 平均 3-4 秒/镜头为黄金节奏
   - 短镜头(<2s)做蒙太奇快切
   - 长镜头(>8s)承载叙事/对话
   - 长短交替，避免节奏单一

3. **内容配比参考**: 美食 30-40% + 风景 30% + 人物/互动 15% + 日常/交通 15%
   （根据素材实际情况灵活调整）

4. **素材使用原则**:
   - 每条素材标注了【推荐有效区间】，优先使用推荐区间
   - 如果叙事需要，可以覆盖推荐区间（如保留某个有趣的口令）
   - 纯画面素材不要超过推荐时长上限
   - 同一素材可以用在不同clip中（不同时间区间）
   - 不是所有素材都要用，果断跳过质量差或重复的

5. **特殊处理**:
   - 精彩片头：从各段落选最有视觉冲击力的镜头做片头蒙太奇（放在你的输出的editing_notes里说明选哪些）
   - 竖屏素材（分辨率标注为竖屏的）：建议跳过
   - 对话/旁白类长素材：保留核心内容，去掉冗余

### 输出格式

请严格按以下JSON格式输出，不要加任何其他文字：

```json
{
  "title": "视频标题（吸引人的，带emoji可选）",
  "structure": [
    {
      "section": "段落名 — 副标题",
      "description": "本段落1-2句话概述",
      "clips": [
        {
          "file": "素材文件名.mp4",
          "start": 0.0,
          "end": 12.0,
          "note": "画面内容简述",
          "subtitle": "保留的语音文字（无语音则空字符串）"
        }
      ]
    }
  ],
  "bgm_suggestion": "BGM风格建议（需包含：genre流派、mood情绪、instruments乐器、tempo快慢、bpm建议值。示例：'genre: cinematic acoustic; mood: warm, nostalgic; instruments: piano, strings, acoustic guitar; tempo: slow; bpm: 80-90'）",
  "editing_notes": "整体剪辑说明，包括片头高光选片建议"
}
```

### 注意事项

- start/end 精确到0.1秒，不要截断语音中间
- note 简洁描述画面（10-30字），用于分镜预览
- subtitle 填入该时间段内的语音文字（来自素材分析中的转写数据）
- section 格式统一为"主题 — 副标题"
- 段落数量通常 8-15 个，太少显得仓促，太多显得碎
- 每个段落 2-7 个clips，保持节奏
